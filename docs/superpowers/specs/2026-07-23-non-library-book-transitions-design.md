# 非书库入口的书籍开合动画规格

日期：2026-07-23
状态：交互设计已批准，待实施

## 1. 目标

为无法稳定捕获封面屏幕位置的阅读入口提供统一、可逆且流畅的动画，同时保留书库页现有的封面展开与缩回动画。

- 书库页继续使用封面几何位置驱动的 `BookOpenAnimation`。
- 首页“继续阅读”使用纸面浮现与退场。
- 发现页从书籍详情进入阅读器时使用底部面板到纸面的方向接力。
- “加入书架”使用原地完成反馈，不伪装成一次阅读路由跳转。
- 所有新动画都不缩放阅读器正文或来源页面。
- 尊重系统的“减少动态效果”设置。

## 2. 非目标

- 不修改书库页现有封面飞行、缩回坐标解析和预测性返回行为。
- 不增加第三方动画依赖。
- 不改变书籍、书源、阅读进度、下载任务或书架数据模型。
- 不让封面飞向底部导航栏；导航顺序可配置且目标可能不可见。
- 不在退出发现页阅读器时自动重新弹出书籍详情面板。

## 3. 统一动画语言

非书库入口统一表达为“纸张进入阅读空间”：

1. 阅读主题背景先成为稳定的全屏底色。
2. 阅读器从下方少量位移到位，同时淡入。
3. 正文比纸面稍晚出现，避免加载内容与系统栏切换抢占首帧。
4. 退出时正文先淡出，纸面向下退场并露出来源页面。

只允许以下属性参与路由动画：

- `opacity`
- `FractionalTranslation` / `SlideTransition`
- 固定纯色背景

禁止对完整阅读器使用 `ScaleTransition`、动态模糊或复杂裁剪，避免正文重绘、字号视觉跳变和低端设备掉帧。

推荐参数：

| 阶段 | 时长 | 曲线 | 位移 |
|---|---:|---|---:|
| 首页进入 | 400ms | `easeOutQuart` | 下方 `0.025` 视口高度 |
| 首页退出 | 320ms | `easeInCubic` | 下方 `0.035` 视口高度 |
| 发现页进入 | 380ms | `easeOutQuart` | 下方 `0.04` 视口高度 |
| 发现页退出 | 300ms | `easeInCubic` | 下方 `0.045` 视口高度 |
| 加入成功反馈 | 320ms | `easeOutBack`，仅作用于图标/小封面 | 8–12 logical pixels |
| 成功后停留 | 500ms | — | — |

减少动态效果开启时，位移归零，路由只做不超过 120ms 的交叉淡入；加入书架只切换勾号和文案。

## 4. 首页“继续阅读”

### 4.1 打开

点击首页主卡或最近阅读条目后：

1. 立即调用现有 `BookOpenTransition.beginActivity()`，让手机悬浮导航收起并屏蔽重复点击。
2. 完成书籍记录读取后，通过无封面几何信息的阅读路由打开阅读器。
3. 路由背景首帧使用阅读主题背景；主题暂不可用时使用当前 `colorScheme.surface`。
4. 阅读器整体从下方轻微上移并淡入，正文透明度使用约 `60–100ms` 的延迟区间。

本地和在线书籍使用同一动画语义，不因阅读器实现不同而表现成两套导航。

### 4.2 退出

- 正文透明度先下降。
- 阅读纸面向下轻移并淡出。
- 首页保持原位置，不缩放、不横移。
- 现有悬浮导航在退出开始时同步回弹。
- Android 预测性返回取消时，阅读页和导航恢复当前进度，不闪回首页。

## 5. 发现页阅读接力

### 5.1 详情面板到阅读器

发现页仍以底部面板展示书籍详情。点击“阅读”后：

1. 阅读按钮进入短暂的按下/忙碌状态，阻止重复提交。
2. 详情面板按自身反向动画向下关闭。
3. 阅读路由从同一纵向轴线进入；背景先铺满，内容从下方轻移并淡入。
4. 面板关闭与阅读路由进入允许有约 `40–70ms` 的交叠，避免中间露出静止发现页形成断拍。

实现应使用一个明确的导航协调方法串联关闭面板和推入路由，避免散落的 `Navigator.pop()` 与 `Navigator.push()` 竞争。

### 5.2 退出

- 阅读器向下轻移并淡出，直接返回原发现页滚动位置。
- 不重新打开详情面板。
- 不刷新发现页列表，除非现有业务数据确实发生变化。
- 预测性返回手势应连续驱动透明度和纵向位移。

## 6. “加入书架”完成反馈

加入书架不是页面导航，使用详情面板内部的状态转换：

```text
BookDetails → ShelfOptions → Submitting → Added / AlreadyAdded / Downloading / Error
```

### 6.1 在线加入

1. 详情内容通过 `AnimatedSwitcher` 切换为“在线加入 / 下载到本地”选项。
2. 用户选择在线加入后，保留当前面板并显示提交状态。
3. 成功时选项行变为勾号和“已加入书架”。
4. 新加入成功时，小封面向下移动 8–12px，并被一条书架横线遮住；该动画只作用于小图层，不移动真实发现页卡片。减少动态效果开启时省略这一步。
5. 停留约 500ms 后，面板向下关闭。

若书籍已经在书架，只显示信息态，不播放“新书落架”动画；短暂停留后关闭。

### 6.2 下载到本地

- 不在选择后立即关闭面板。
- 内容切换为下载任务状态，并提供“后台下载”退出动作。
- 后台下载后使用现有侧边提示，不播放“已加入书架”成功动画。

### 6.3 失败

- 面板保持打开。
- 显示可理解的错误状态并允许重试。
- 不自动关闭，也不播放勾号或落架动画。

## 7. 代码边界

### 7.1 阅读路由

扩展 `BookOpenTransition.createRoute` 的无封面兜底路径，使其接收入口语义：

```text
BookReaderTransitionOrigin
  ├─ home
  └─ discoverSheet
```

入口语义只控制时长、纵向位移和透明度区间；活动计数、悬浮导航、预测性返回、`route.completed` 等生命周期继续由现有 `BookOpenTransition` 统一管理。

### 7.2 首页入口

首页打开方法负责传入 `home` 入口语义，不复制路由动画代码。本地阅读仍通过 `NativeReaderService`，在线阅读仍通过 `BookSourceReaderPage`。

### 7.3 发现页详情

`SourcedBookActions` 负责单个底部面板中的详情、加入方式和结果状态。业务服务保持不变，组件只协调视觉状态和导航时序。

## 8. 可访问性与性能

- 使用 `MediaQuery.disableAnimationsOf(context)` 判断减少动态效果。
- 动画期间按钮进入忙碌态，防止重复打开阅读器或重复加入书架。
- 阅读器路由保持 `allowSnapshotting: false`，确保预测性返回和加载态正确。
- 路由只动画合成友好的透明度与位移；不添加实时 blur、全屏 shader 或正文 scale。
- 面板关闭后再访问其 `BuildContext` 前必须检查 mounted 状态。

## 9. 验收用例

1. 从首页主卡打开本地书：无缩放，纸面上浮淡入，退出返回首页原位置。
2. 从首页打开在线书：与本地书使用相同进入/退出语义。
3. 首页快速连点：只产生一个阅读路由。
4. 从发现页详情点击阅读：面板与阅读器纵向接力，中间无白屏或明显停顿。
5. 从发现页阅读器退出：直接回到发现页原滚动位置，不重新弹出详情。
6. Android 预测性返回取消：阅读器恢复，悬浮导航不错误显示。
7. 在线加入成功：显示勾号和完成态，停留后面板向下关闭。
8. 重复加入：显示“已在书架”，不播放新书落架动画。
9. 加入失败：面板保留并可重试。
10. 下载到本地：显示下载状态，可转后台，不提前播放完成反馈。
11. 减少动态效果开启：无位移和缩放，只保留短淡入与状态切换。
12. 书库网格、卡片和列表入口：现有封面展开/缩回动画完全不变。

## 10. 验证范围

- 为无封面阅读路由补充进入、退出、减少动态效果和预测性返回回归。
- 为首页入口补充入口语义和重复点击回归。
- 为发现页详情补充面板接力、直接返回发现页、在线加入成功/重复/失败和下载状态回归。
- 运行目标测试、`flutter analyze --no-pub`，再运行全量 Flutter 测试。
- 在 Android 真机检查首页、发现页、系统返回和预测性返回；iOS 检查交互式返回不可用时的普通退场。
