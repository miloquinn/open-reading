# 阅读页独立上下边距设计

## 目标

将原有单一“上下边距”设置拆分为“上边距”和“下边距”，让用户分别调整正文顶部与底部留白。默认布局应贴近屏幕，但仍自动避开灵动岛、刘海和系统手势区域。页码位置独立于正文下边距，并在 iOS 上进一步靠近屏幕底部。

## 设置与默认值

- 上边距滑杆：`0–40pt`，步进 `1pt`，默认 `4pt`。
- 下边距滑杆：`0–40pt`，步进 `1pt`，默认 `0pt`。
- 上边距的最终值为系统顶部安全区加用户上边距。
- 下边距的最终值为系统底部安全区、用户下边距以及页码避让空间三者所需的最小安全值，不重复相加。
- 页码不随下边距滑杆移动；它单独根据系统底部安全区定位。

## 页码位置

页码下沿使用统一安全区公式：`max(8pt, systemBottomInset - 20pt)`。在当前 SloanePro 的 `34pt` 底部安全区上，页码下沿约为 `14pt`，比现有约 `20pt` 更靠近底部，同时保持在 Home Indicator 上方。无底部安全区或不同平台的设备仍使用同一公式，不按机型或操作系统写分支。

## 持久化与迁移

新增独立的上边距、下边距偏好键。首次读取新版本设置时：

- 若已有新键，直接使用并限制到 `0–40pt`。
- 若只有旧的单一上下边距值，将旧值相对旧默认值 `28pt` 的增量迁移到两个新默认值：上边距为 `4 + 增量`，下边距为 `0 + 增量`。
- 旧值为默认 `28pt` 时，迁移结果正好是新的 `4pt / 0pt` 默认值；用户过去主动增加的边距仍会保留。
- 保存新设置后不再写入旧键，但保留旧键读取逻辑用于一次性迁移。

## 代码边界

- `NativeReaderPage` 负责加载、保存、预览两个独立设置，并在设置面板显示两个滑杆。
- `ReaderSafeAreaMetrics` 只负责把系统安全区、用户边距和页码位置转换为最终布局指标。
- 分页缓存签名同时包含最终顶部与底部布局值，任一滑杆变化都会触发正确的重新分页。
- 本次只调整原生阅读页，不改变书源阅读器的设置模型。

## 文案

原“上下边距”替换为“上边距”和“下边距”。为现有支持语言补齐对应本地化键，避免设置页出现硬编码中文。

## 验证

- 单元测试覆盖 iPhone、Android、无底部安全区设备以及旧设置迁移。
- 阅读器布局测试验证上下边距分别进入缓存签名并触发重新分页。
- `flutter analyze --no-pub` 无新增问题。
- Profile 构建并覆盖安装到 SloanePro，真机截图验证：正文避开灵动岛、页码约距底部 `14pt`、两个滑杆可独立调整且重启后保留。

## 验收标准

1. 设置面板显示独立的上边距和下边距滑杆。
2. 默认正文比旧版本更贴近屏幕，上方不被灵动岛遮挡。
3. 调整上边距不改变下边距，调整下边距不改变上边距或页码位置。
4. 页码在 SloanePro 上比当前版本更靠近底部且不与 Home Indicator 重叠。
5. 不存在按 iPhone 型号、iOS 或 Android 写死的布局分支。
