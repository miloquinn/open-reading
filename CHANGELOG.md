# Open Reading 更新日志

本文件记录 Open Reading 各公开版本的主要变化。版本按发布时间倒序排列。

## v2.3.6 - 待发布

### 在线书源与阅读连续性

- 已读书源的目录和正文会优先从本机缓存恢复；目录超过 30 分钟、正文超过 12 小时后在后台刷新，30 天内缓存可在书源变慢或离线时继续阅读。整本下载仍会强制获取最新正文。
- 首页打开在线书籍时会直接进入书源阅读器，不再按本地文件处理。
- 在线书源的水平滑动模式会提前下载并分页相邻章节，把前后章节的全部页面接入同一条连续翻页序列；章节标题与正文之间可像普通页面一样连续前后翻动，快速连续翻页或重新接管动画时不再在章节边界回弹。
- 修复夜间、纯黑和自定义深色阅读主题打开书籍时的短暂闪白，动画与阅读器加载首帧统一使用当前阅读主题背景。

### 在线字体下载更流畅

- 大型在线字体的文件签名和 SHA-256 完整性校验改在后台执行；下载进度只刷新字体列表，不再反复重建应用根界面与设置页。

### 书库网格阅读信息

- 书库网格新增“显示书名和进度”开关，默认关闭并继续保持纯封面布局。
- 开启后，每本封面下方显示单行书名、精简进度条和百分比；过长书名自动以省略号截断，网格行高保持一致。

### WebDAV 原文件与可读目录

- 新上传的书籍以未加密的原始字节、原文件名和扩展名保存到 `books/<书名 - 作者>/<原文件名>`；SHA-256 只保留在同步元数据和本地索引中用于完整性校验。
- 同名同内容文件直接复用，同名但内容不同的书使用 `(2)`、`(3)` 可读编号，避免覆盖已有文件。
- 旧版 `blobs/books/sha256/...` 文件继续正常下载，无需重新同步；检测到旧记录时，书籍文件页面会显示兼容说明。

## v2.3.5 - 2026-07-22

正式版本：`2.3.5`，基础构建号：`260722009`。

### 更直接的首页

- 首页重新聚焦“继续阅读”：最近一本书直接展示封面、作者、阅读进度和继续阅读入口，其他最近阅读以轻量封面列表呈现。
- 今日、本周、累计阅读时长与七日阅读节奏合并为一张统计卡；移除首页的每日目标、阅读计划、专注计时和 AI 阅读建议，不再为首页初始化对应服务。
- 手机与平板、桌面端统一复用同一套响应式首页实现，并补充 320px 手机与 1280px 宽屏布局回归。

### 浏览器版本与自动部署

- 浏览器端数据库接入基于 IndexedDB 的 SQLite 兼容层，并跳过仅适用于本地文件系统的图片目录、路径修复和应用内更新检查，使 Flutter Web 可以稳定启动并保留结构化数据。
- GitHub Release 流水线新增 Flutter Web 构建、版本元数据校验和内部部署产物；发布后通过受限 SSH 账号原子切换官网 Web 版本，并校验归档摘要与线上 `version.json`。
- Web 部署保留最近版本目录，上传、解包和软链接切换均限制路径、所有者、权限和归档内容，失败时不会覆盖当前线上版本。

### 平台发布与界面细节

- macOS 发布流水线支持 arm64 / x86_64 Universal App、Hardened Runtime、Developer ID 签名、Apple 公证、staple 和 Gatekeeper 验证；正式功能仍由受保护发布环境显式启用。
- 删除本地书籍导入时残留的联网封面搜索；EPUB/PDF 优先使用文件内封面，TXT/MOBI 等无封面格式直接在本地生成默认封面。
- 设置页移除与阅读无关的开发者产品推广，并压缩 AI 快捷模型卡片的高度、间距和状态表达，让设置入口更集中。
- 统一书库纯封面网格的裁切规则，并继续调整 Android 启动图标安全区和 iOS 旧系统回退资源的视觉比例。

## v2.3.1 - 2026-07-22

Android 版本号：`2.3.1`，基础构建号：`260722005`；iOS TestFlight 构建号：`260722007`。

### WebDAV 稳定性与选择权

- 当前同步范围聚焦书架信息、阅读进度、书签和阅读统计；笔记/高亮暂不出现在同步界面，但保留稳定协议身份和未来幂等迁移路径。
- 新导入书籍支持“每次询问（默认）/ 自动上传 / 始终手动”三种文件策略，不会因为开启 WebDAV 就静默上传全部书籍。
- 增强 Android 文件临时不可访问和 WebDAV HTTP 409 兼容处理，并在失败时显示连接、扫描、读取、合并或上传等具体阶段。
- 书籍文件同步会同时传输持久封面；跨设备恢复后以同步元数据校正书名和作者，不再把 TXT 第一章标题误当书名，也不会使用错误的本机重建封面。
- 连接配置与同步内容拆分为独立页面；阅读进度等范围开关修改后立即保存并从下一次同步生效。

### 缓存与书源封面

- 新增安全缓存管理，只清理书源封面、在线章节和临时更新文件，不触碰书籍、进度、数据库、设置或凭据。
- 书源封面使用并发受限、可重试的内存与磁盘缓存；加入书架时保存持久封面，损坏缓存可自动驱逐并重新获取。

### 下载任务取消

- 下载任务列表和下载进度弹窗新增单任务取消；等待中的任务不会开始，下载中的任务会终止当前网络请求、移除前台通知并继续处理下一条队列任务。
- 取消状态与下载失败分开显示，取消发生在文件落盘阶段时会清理未完成文件。

## v2.3.0 - 2026-07-22

Android 版本号：`2.3.0`，基础构建号：`260722003`；iOS TestFlight 构建号：`260722004`。

### ColorOS 流体云与实时下载

- 支持 ColorOS 流体云：书籍与官网 APK 下载过程中，可在流体云和状态栏持续显示实时进度与百分比短状态，离开当前页面后仍能查看下载状态。
- 底层使用 Android 16 原生实时进度通知并请求 promoted ongoing 展示；系统版本、设备或 OEM 未提供流体云展示时自动回退为普通进度通知，不影响后台下载。

### WebDAV 跨设备同步

- 新增本地优先的 WebDAV 同步，可将书架信息、阅读进度、书签、笔记、高亮与阅读统计同步到用户自己的 WebDAV 空间；支持手动同步以及应用启动、回到前台时自动同步。
- 离线修改会先可靠保存在本机，网络恢复后继续上传；多设备变更使用稳定设备身份、混合逻辑时钟、删除标记和记录级合并，避免简单整库覆盖造成数据丢失。
- 新增同步状态、待处理数量、最近同步时间、同步范围、活动结果和分类错误提示；损坏的远端批次、异常时钟偏差或不满足安全要求的服务不会被静默应用。

### 书籍原文件同步与连接安全

- WebDAV 书籍文件传输采用明确授权和逐本选择：只有开启“允许上传书籍原文件”并主动选中的书才会上传；其他设备可查看远端可用文件并按需下载。
- 书籍文件按 SHA-256 内容寻址并去重，当前单文件安全上限为 100 MiB；远端只有书架记录、尚未下载原文件时会明确显示“云端”状态，不伪装成本地可读文件。
- WebDAV 密码只保存在当前设备的安全存储中；连接默认要求 HTTPS，并限制凭据重定向到同一服务源。界面明确说明首版不是端到端加密，WebDAV 服务提供方仍可能读取远端内容。

### 书籍导出、文件关联与系统分享

- 书库本地书支持长按导出：Android 默认复制到公共 `Download/开元阅读/`，iOS 与桌面端使用系统保存面板；导出不会改变书库原文件和数据库路径。
- Android 可从文件管理器或其他应用直接打开、单文件分享或多文件分享 TXT/EPUB；单本导入后直接阅读，多本进入导入队列。iOS、macOS 与 Linux 接入各自系统文件打开入口，Windows/Linux 启动参数进入相同安全导入链路。
- 外部文件先物化到受控暂存区，再执行扩展名、MIME、文件头、数量和大小校验；冷启动、热启动与异常恢复统一按持久化 FIFO 处理。

### 书库布局

- 书库新增“卡片 / 网格”布局切换：卡片继续展示书名、作者和阅读进度，网格专注显示封面。
- 手机网格可选择每行 2 本或 3 本；平板和桌面端按相同封面密度响应式增加列数，旋转窗口后自动重新排布。

### 书源管理界面

- 重做手机端书源卡片布局，名称、描述、能力标签、启停状态和操作按钮分层排列，长名称和大量能力标签不再被横向控件挤到无法阅读。
- 手机端“添加书源”改为键盘自适应的全宽底部面板，地址、第三方责任说明、确认勾选和连接操作拥有独立空间；平板与桌面端继续使用宽屏对话框。

### 无阻塞应用提示

- 更新检查、用户协议、书源、字体、同步等页面的短反馈统一改为顶部浮层；手机顶部居中、宽屏右上显示，连续提示直接替换并自动退场。
- 提示浮层支持点击穿透、无障碍 live region 和减少动态效果，不再用底部提示条遮挡阅读或页面操作。

## v2.2.7 - 2026-07-22

Android 版本号：`2.2.7`，基础构建号：`14138`。

### 全格式首行缩进修复

- 修复 TXT 中 Unicode 换行符只产生视觉换行、没有被阅读布局识别为段落边界的问题；CR/LF、VT、FF、NEL、Unicode 行分隔符和段落分隔符现在使用同一套段首判断。
- EPUB 的 `<br>` 与 `div`、`section`、`article`、`stanza` 等正文容器统一转换为明确段落边界，避免部分段落粘连或遗漏缩进；HTML/HTM/XHTML、Markdown、FB2、RTF、DOCX 与在线书源同步进入同一套首行缩进管线。
- 段首已有的普通空格、NBSP、Unicode 空格、全角空格和 BOM 会先规范化，再按阅读设置生成一致的视觉缩进；书签、进度和高亮仍使用原文 canonical offset。

### 开放书源协议与分类浏览

- Open Reading Source Protocol 升级到 `1.4`，书源必须声明 `search`、`detail`、`catalog`、`content` 四项核心阅读能力；协议文档、OpenAPI、示例书源和应用内说明保持一致。
- 书源分类由同步铺开的全部标签改为可搜索的惰性选择器；移动端使用底部面板，宽屏使用对话框，并按书源分组，分类数量较多时不再一次构建所有选项。
- 已添加书源支持手动刷新发现清单，在保留本地启用状态和添加时间的同时更新名称、描述、能力与服务地址；清单不得静默改变书源稳定 ID。

### 应用内更新日志

- 应用内版本历史改为从统一 JSON 资产异步加载，四语内容、版本顺序和回退规则集中维护；页面增加加载失败提示和重试操作。

## v2.2.6 - 2026-07-22

Android 版本号：`2.2.6`，基础构建号：`14137`。

### 下载任务与后台进度

- 在线书籍下载改为共享任务队列，关闭进度弹窗或离开当前页面后仍会继续；书库新增下载任务入口，可统一查看排队、进度、完成与失败状态。
- Android 的书籍和应用更新下载会通过前台服务显示系统进度通知；完成通知可直接打开书籍或进入已校验 APK 的安装流程，通知权限或通知桥失败不会中断下载。

### 在线阅读修复与流畅度

- 修复在线阅读部分段落首行缩进失效的问题：书源 HTML 正文若仅以裸换行分隔段落（未使用 `<p>`/`<div>`/`<br>`），且中间穿插了图片等内联标签，此前换行会被段内空白折叠规则一并吞掉，导致这些段落被拼接成一整段、只有开头能获得首行缩进；现在裸换行统一按段落边界处理，同一章节内的所有段落缩进表现一致。
- 优化在线书源的水平跨章切换：优先预取下一章并复用已预热分页布局，动画结束后直接衔接目标页，避免停稳后出现可见的页面控制器重置。

### Android 图标

- 进一步缩小 Android 普通图标与自适应图标中的书本主体，保持原有品牌图案和配色，同时增加四周留白，改善圆角方形与圆形启动器中的视觉比例。

### 字体资源改为在线下载

- App 界面字体与阅读正文字体不再随安装包一起分发，改为首次使用时按需从 GitHub 在线下载，安装包体积明显减小；已下载字体会保留在本机，删除下载后下次使用会自动重新获取。

### AI 阅读设置与错误提示

- 完善 AI 阅读设置：支持配置多个模型、预设模型快速选择、自动获取可用模型列表，并可分别设置 Base URL、API Key 和 Temperature 等参数。
- AI 服务、语音朗读、书注翻译、阅读计划等模块的失败提示统一改为本地化文案，替代原始异常信息。

## v2.2.5 - 2026-07-20

Android 版本号：`2.2.5`，基础构建号：`14134`。

### 文字阅读内核统一

- 本地 TXT/EPUB 文本与在线书源章节统一使用同一套 canonical text、显示投影、分页页面模型、正文尺寸计算和最终 `RichText` 渲染，避免相同设置在不同入口产生不同换行与分页。
- 在线章节标题改为与结构化 TXT 一致的独占标题页；书源 HTML/纯文本适配器不再预先注入全角缩进或破坏纯文本空行，首行缩进和段间距统一由阅读设置生成。
- 修复部分在线 TXT 章节在独占标题页后，正文第一页仍重复显示同一标题的问题；仅移除正文开头与接口标题或目录标题完全匹配的首行/首段，不影响正文中正常出现的标题文字。

### 在线阅读流畅度

- 修复水平滑动到下一章时在动画过半就重建 `PageView` 导致的顿挫；现在先使用预取和预热完成的真实目标页结束整段横滑，停稳后才提交章节状态，中途回滑不会误切章。
- 下一章内容、分页布局和相邻页预览继续提前准备，跨章提交不等待阅读进度持久化。

### 应用图标

- 更新 Android、iOS、Windows、macOS、Linux、Web 与 OpenHarmony 图标资源，统一各平台的正式品牌视觉和安全区表现。
- 收窄手机首页悬浮导航栏的水平尺寸，增加两侧呼吸空间，同时保持图标、文字模式和点击区域清晰可用。

- 修复目录/书签弹窗的章节列表在 `StatefulBuilder` 构建闭包内每帧重新分配的问题：该闭包会在键盘弹出/收回动画的每一帧被重新调用，此前每帧都会为全部章节重新创建对象；改为仅在打开面板时构建一次，章节数越多的书受益越明显。
- 修复阅读页在目录搜索框弹出键盘时整页重新排版的问题：阅读页 `Scaffold` 默认会为任意来源的键盘让位（含弹窗内输入框触发的键盘），导致排版区域尺寸在键盘动画的每一帧都发生变化，从而触发当前章节的整页重新分页；阅读页本身没有输入框，现在关闭该让位行为。

### EPUB 排版修复

- 修复部分 EPUB 首行缩进无效／缩进出现在段落中间的问题：源文件中 `<p>`、标题、列表项等段落文本若在 XHTML 里被折行排版，其中的裸换行此前会被误判为新段落，导致缩进插在折行处而非每段真正的开头；现在统一将段落内部空白（含换行）折叠为空格，仅 `<pre>` 保留原始换行。

### Android 图标

- 缩小 Android 自适应图标前景中的书本图形占比（约 80%，四周留出安全边距），避免图标被系统蒙版裁切成圆形/圆角方形等形状时显得比其他 App 图标更大更满；背景渐变保持通栏铺满，边缘无缝衔接。
