# Open Reading 更新日志

本文件记录 Open Reading 各公开版本的主要变化。版本按发布时间倒序排列。

## v2.2.4 - 2026-07-20

Android 版本号：`2.2.4`，基础构建号：`14133`。

### 开放书源协议 1.3

- App 支持的 Open Reading Source Protocol 版本统一为 `1.3`，协议说明、OpenAPI、示例发现文档、参考书源和应用内说明保持一致。
- 章节目录支持按 `page`、`pageSize` 和 `hasMore` 完整翻页拉取，避免长篇书籍只显示首批章节。
- 客户端遵守书源声明的 `maxCatalogPageSize`；未声明时使用协议默认值，并继续兼容旧书源一次返回完整 `items` 的响应。

### 在线阅读与界面修复

- 修复书源 HTML 正文以 `<br>` 分隔段落时丢失段落边界的问题。
- 缩小手机首页悬浮导航栏高度，减少对内容区域的占用。

## v2.2.3 - 2026-07-20

Android 版本号：`2.2.3`，基础构建号：`14132`。

### 阅读器目录性能

- 修复阅读器目录/书签面板首次打开卡顿、搜索输入及键盘弹出收回时掉帧明显的问题。
- 章节树与搜索过滤结果改为按需缓存，仅在章节数据、搜索关键词或章节展开／折叠状态变化时才重新计算。
- 修复目录/书签弹窗的章节列表在 `StatefulBuilder` 构建闭包内每帧重新分配的问题：该闭包会在键盘弹出/收回动画的每一帧被重新调用，此前每帧都会为全部章节重新创建对象；改为仅在打开面板时构建一次，章节数越多的书受益越明显。
- 修复阅读页在目录搜索框弹出键盘时整页重新排版的问题：阅读页 `Scaffold` 默认会为任意来源的键盘让位（含弹窗内输入框触发的键盘），导致排版区域尺寸在键盘动画的每一帧都发生变化，从而触发当前章节的整页重新分页；阅读页本身没有输入框，现在关闭该让位行为。

### EPUB 排版修复

- 修复部分 EPUB 首行缩进无效／缩进出现在段落中间的问题：源文件中 `<p>`、标题、列表项等段落文本若在 XHTML 里被折行排版，其中的裸换行此前会被误判为新段落，导致缩进插在折行处而非每段真正的开头；现在统一将段落内部空白（含换行）折叠为空格，仅 `<pre>` 保留原始换行。

### Android 图标

- 缩小 Android 自适应图标前景中的书本图形占比（约 80%，四周留出安全边距），避免图标被系统蒙版裁切成圆形/圆角方形等形状时显得比其他 App 图标更大更满；背景渐变保持通栏铺满，边缘无缝衔接。

## v2.2.0 - 2026-07-19

Android 版本号：`2.2.0`，基础构建号：`12119`。

### 更新与官网分发

- 更新检查同时查询 GitHub Releases 与官方站点；发现新版本后可明确选择从 GitHub 更新，或从官网下载。
- Android 官网更新会在应用内按设备 ABI 下载对应 APK，只接受官网 HTTPS 地址及同域重定向，并以 512 MiB 为硬上限在下载过程中即时拦截超出声明大小的响应；下载完成后校验 SHA-256，并在原生层复核包名、实际版本码及当前应用签名后交给系统安装器。Android 8 及以上会按需请求“安装未知应用”授权，最终安装仍由用户确认。
- iOS 当前从官网选项打开网页，待正式上架后再切换 App Store；Windows、Linux 与其他平台继续打开外部下载页。
- GitHub Tag 发布成功后会把 Android、Windows、Linux 安装包和校验清单自动镜像到 `open.xxread.top`；导入前复核固定仓库与 Tag、完整资产集合、校验和、APK 包名、实际版本码和正式签名，官网 stable latest 按平台与架构保持版本单调。
- 官网后台记录真实下载短链的版本、架构、时间、IP 和 User-Agent，用于下载次数、独立 IP、安全防护与故障排查；原始 IP 下载明细最多保留 30 天，生产凭据、数据库、安装包和访问日志均不进入 Git。

### 平板双页阅读

- 阅读设置新增仅平板可见的“双页布局”开关，手机端不显示该选项；关闭后恢复单页并保持当前阅读位置。
- 平板横屏双页时，左页左上只显示章节标题，右页右上只显示时间和电量，避免状态信息重复。
- 修复仿真翻页的跨书脊层级：翻到下一页时右页正确覆盖左页，翻到上一页时左页正确覆盖右页；空白与补位页不再显示虚假页码。

### 仿真翻页与阅读性能

- 重做上一页与下一页的手势跟随、折线几何和回弹收尾，减少纸张脱手、中部起手闪跳、右下甩尾和终点阴影复现。
- 优化平板书脊、中缝和空白补页，本地文件与在线书源使用一致的双页步进、跨章和进度保存逻辑。
- 本地 TXT 阅读页延后到封面转场完成后再初始化，降低打开书籍时的卡顿。
- 在线书源优化相邻章节预取、后台进度保存和分页复用，跨章翻页不再等待较远章节或持久化完成。

### 界面与个性化

- 手机底部悬浮导航支持“纯图标”和“图标＋文字”两种模式，并优化选中状态、尺寸、标签动画与按压反馈。
- 预设与自定义阅读主题可以统一拖拽排序并自动保存；修复主题卡片选中描边被背景遮挡或断裂的问题。
- 阅读统计详情页升级为统一的书卷风格，重新设计总览、图表、热力图、书籍排行和成就页面。
- 时间、章节标题、电量和页码继续作为纸页的一部分，随横向滑动和仿真翻页一起移动。

### 工程结构与可维护性

- `lib/pages` 按首页、书库、书源、阅读器、阅读统计、设置和协议七个功能域重新归档，页面、控制器、主题、组件和私有 `parts` 就近维护。
- 页面文件统一采用 `*_page.dart`、`*_controller.dart`、`*_widgets.dart` 和 `*_part.dart` 命名规则，跨功能域引用统一使用包路径，公开页面 API 保持兼容。

### 第三方书源与项目支持

- 官方发行版明确不预装、不内置、不推荐任何第三方书源；欢迎页和添加书源流程新增独立责任确认。
- 新增 `SOURCE_POLICY.md`，说明项目、独立书源运营者和用户各自的责任边界。
- 设置页新增开发者产品与完全自愿的支持入口；新用户完成欢迎页后会显示一次性项目支持引导。

- “全部 / 在读 / 已读”筛选收纳到顶栏菜单。
- 修复玻璃效果下书架顶栏缺少毛玻璃质感的问题。

### 阅读设置

- 修复阅读设置面板无法通过顶部拖动区域下拉关闭的问题。
- 统一本地阅读器和在线书源阅读器的设置入口、翻页模式与字号范围。
- 字号、行距和边距在结束拖动后再应用排版，减少拖动卡顿。

## v1.0.1

### 发现与书源

- 新增发现体验，支持书源推荐、分类和最新内容浏览。
- 搜索结果支持自动加载下一页、手动加载更多和失败重试。
- 多书源分页独立维护状态，并在聚合结果时自动去重。

### 开源许可

- 从本版本起采用 GNU AGPL-3.0-only。
- `v1.0.0` 及更早版本已经获得的 MIT 权利继续有效。
- 新增授权边界、贡献许可和项目品牌说明。
- 更新用户协议与多语言许可文案。
